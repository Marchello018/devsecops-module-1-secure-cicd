# Модель угроз CI/CD

## Активы
- Исходный код
- Манифест зависимостей
- Конфигурация pipeline
- Docker image и build-артефакты
- GitHub token и метаданные репозитория

## Границы доверия
- Pull request из fork считаются недоверенным контекстом
- Push в `main` считается доверенным после review
- Production deployment требует защищённого environment

## Угрозы
- Утечка секретов через логи, debug output или небезопасные actions
- Злоумышленный pull request с выполнением атакующего кода
- Компрометация стороннего GitHub Action
- Избыточные права у `GITHUB_TOKEN`
- Подмена артефакта между сборкой и деплоем
- Злоупотребление runner через зависшие или долгие job

## Матрица угроз

| Угроза | Влияние | Контроль |
| --- | --- | --- |
| Утечка секретов в логах или actions | Раскрытие учётных данных и дальнейшая компрометация | Не передавать секреты в PR-контекст, аккуратно вести логи, ограничивать права токена |
| Злоумышленный PR из fork | Выполнение произвольного кода в CI | Считать fork PR недоверенным, не делать deploy из PR, не давать привилегированные секреты |
| Компрометированный third-party action | Supply-chain компрометация внутри workflow | Фиксировать версии actions и минимизировать внешние зависимости |
| Избыточные права `GITHUB_TOKEN` | Изменение репозитория, workflow или артефактов | `permissions: contents: read` и least privilege по умолчанию |
| Подмена артефакта | Деплой не того, что проходило проверки | Разделять стадии package и deploy, оставлять deploy под контролем |
| Злоупотребление runner | Персистентность, лишняя нагрузка или lateral movement | Hosted runners, таймауты, короткоживущие job, защищённый deploy stage |

## Контроли
- Least privilege через `permissions: contents: read`
- Зафиксированные версии actions вместо плавающих тегов
- `persist-credentials: false` на checkout
- Отсутствие deployment-шагов в pull request контексте
- Разделение pipeline на `test`, `scan`, `package` и `deploy`
- Таймауты job и `fail-fast` стратегия
- Защищённый `production` environment с ручным approval

## Остаточные риски
- Hosted runner всё ещё остаётся общей средой выполнения
- Pinning версий снижает, но не устраняет supply-chain риск полностью
- Реальный production deploy всё равно требует безопасной работы с секретами
